=== GAP 3: API ENDPOINT FOR VALUE DATA ===

Add to server/routes.ts:

PUT /api/use-cases/:id/value
  Request body: {
    investment: {
      initialInvestment: number,
      ongoingMonthlyCost: number,
      currency: "GBP"
    },
    selectedKpis: string[],
    kpiValues: {
      [kpiId]: {
        baselineValue: number,
        targetValue: number,
        isOverridden: boolean,
        overrideReason: string
      }
    }
  }
  
  Behavior:
  1. Validate input
  2. Calculate derived values (maturity, ranges) using existing logic
  3. Calculate estimated value using existing formulas
  4. Update use_cases.valueRealization JSONB
  5. Return updated use case with calculated metrics

Also ensure existing PUT /api/use-cases/:id saves valueRealization if included in body.
